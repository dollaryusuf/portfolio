Case study
The problem
Users searched. They didn't find. They left.
SwiftCart's product catalogue had grown organically for four years. Categories reflected the merchant's warehouse logic — not how customers thought. Funnel data showed a 58% abandonment rate at the category browse stage, and internal search queries were being routed to zero-result pages at an alarming rate.
The research / data
Annotating 14,000 search queries to find the real mental model.
Drawing on my data annotation background, I treated 14,000 months of search logs as a labelling task — tagging each query by intent, semantic cluster, and the category the user landed on versus what they actually needed. This surfaced 23 critical mismatches between existing category labels and user language. Affinity mapping then collapsed the merchant's 87-node taxonomy into a 32-node customer-first architecture.
The solution
A taxonomy that speaks user, not warehouse.
New category labels were derived directly from high-frequency query language. A contextual search layer surfaces synonym matches so "trainers" and "sneakers" both resolve correctly. Progressive filters were sequenced in the order users actually narrow down choices — type first, then price, then brand — eliminating the paralysis of a 12-filter sidebar on first load.
−41%
Reduction in search-to-cart abandonment within 60 days of launch
SwiftCart · Mobile E-commerce Redesign
The problem
Buyers stalled. Not because of price — because of doubt.
SkillSwap's marketplace had healthy seller supply but a conversion problem: buyers browsed extensively and dropped off at the seller profile page. Exit surveys pointed to trust, not cost. Without a formal verification layer, buyers couldn't distinguish a reliable professional from a new account with a polished bio.
The research / data
QA methodology applied to trust signal mapping.
Leveraging my experience in fact-checking and quality assurance, I audited 200 seller profiles against the questions buyers asked in support tickets. I built a trust signal matrix — scoring each profile element (reviews, credentials, response time, portfolio) by its statistical correlation with completed bookings. This replaced gut instinct with an evidence-ranked hierarchy of what buyers actually needed to see, and in what order.
The solution
A verification UI that answers the buyer's unspoken questions.
The redesigned seller profile leads with a verification badge system — three tiers (ID verified, Skill assessed, Top-rated) each with a tooltip explaining the vetting process. Response time and acceptance rate are shown as live stats, not buried in a tab. Review excerpts are surfaced by recency and specificity, replacing star averages with real signal. Every element ordered by the trust matrix score.
+25%
Increase in completed peer-to-peer bookings within 90 days of launch
SkillSwap · Service Marketplace Redesign
The problem
The dashboard was built for analysts. Users were first-time investors.
ZenithPay's existing dashboard displayed real-time market data at full resolution — ticks, percentage swings, asset correlation matrices. Power users loved it. First-time investors opened it, felt overwhelmed within seconds, and closed it without transacting. Time-to-first-transaction for new users averaged 11 minutes; the target was under 3.
The research / data
Heuristic evaluation as a cognitive load audit.
I ran a structured heuristic evaluation against Nielsen's 10 principles, scoring each dashboard element for cognitive demand. 34 of 61 elements violated "recognition over recall" or "aesthetic and minimalist design." Session recordings confirmed users' gaze fixated on the most visually complex element first — a 6-line asset correlation chart — rather than the primary action: placing a trade. Every element was scored by decision-relevance for the casual investor persona.
The solution
Progressive disclosure — complexity on demand, clarity by default.
The default view was rebuilt around one question: "What do I do next?" Portfolio value, daily change, and a single recommended action dominate the viewport. All high-frequency data (tick charts, correlation matrices) was moved behind a "Details" toggle — present for those who want it, invisible for those who don't. Colour encoding was standardised: green/red used only for gains/losses, eliminating the mixed-meaning palette that caused scanning errors.
−68%
Reduction in time-to-first-transaction for new investors (11 min → 3.5 min)
ZenithPay · Fintech Dashboard Redesign
The problem
Designing for 'Intelligence' at speed — the blank canvas problem in AI interfaces.
Most AI dashboards either overwhelm executives with raw model output or hide the intelligence behind so many filters it's unreachable. The hackathon brief demanded a tool that felt smart from the first second — no configuration, no onboarding, no blank canvas paralysis. The constraint: the entire design-to-prototype pipeline had to close within a single sprint day.
The research / data
Lightning 5-minute stakeholder interviews and prompt-tuning as a design tool.
With no time for formal research, I ran structured 5-minute interviews with four surrogate stakeholders — one question each: "What decision did you fail to make last week because you lacked the right data?" Answers clustered into three intent types: situational awareness, anomaly detection, and trend projection. I then used iterative prompt engineering directly as a design tool, testing AI output formats against each intent type until the responses mapped cleanly to UI components — treating every prompt iteration as a low-fidelity wireframe.
The solution
'Insight First' architecture with modular Cognitive Blocks.
Core Intelligence opens on a single synthesised executive briefing — no setup required. Beneath it, the UI is composed of modular 'Cognitive Blocks': self-contained insight cards that each answer one executive question (What changed? Why? What's next?). Blocks reorder dynamically by urgency score, so the most decision-critical insight always leads. A persistent prompt rail lets executives interrogate any block in natural language, collapsing the gap between question and answer to a single interaction. Live at core-intelligence-jet.vercel.app.

AI & Research

Why 5 years of AI work makes me a sharper designer.

Training AI systems isn't removed from design — it's a different lens on the same problem: how do humans understand systems, and how do systems understand humans? Every skill below transfers directly.

Human-in-the-loop design

Years spent making AI outputs explainable to human reviewers trained me to surface system reasoning — a discipline I carry into every UI. If a user can't tell why something happened, the interface has failed.

Bias recognition

Annotating data across four languages — English, French, Arabic, Yoruba — sharpened my eye for underserved edge cases. I apply the same lens to user research: who is missing from the sample, and what does the design assume about them?

Structured evaluation

RLHF requires scoring outputs against precise rubrics — not vibes. I run UX testing the same way: defined success criteria, calibrated reviewers, and documented reasoning so findings survive a handoff.

Prompt engineering as UX

Writing prompts that produce consistent, accurate responses is an exercise in anticipating misunderstanding. The same cognitive discipline shapes how I write microcopy, empty states, and error messages — every word is a prompt the user gives the system.
