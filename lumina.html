The Why

The Hybrid Edge

Lumina's interface decisions aren't aesthetic guesses — they're informed by years spent inside the machinery that fine-tuning tools are meant to expose.

Five years in RLHF taught me where fine-tuning tools actually break down: not in the math, but in the moment a creator has to guess what an invisible parameter is doing to their output. Annotators and reward-model teams live with that ambiguity daily — Lumina is built to remove it.

My background in Computer Science means every "simple" control in Lumina — a slider, a toggle, a comparison view — is grounded in what's actually happening underneath: sampling distributions, token probabilities, loss curves. The interface never promises more certainty than the model can give.

That combination produced Lumina's central bet: complexity should be visible, not hidden — but only when the creator asks for it.

Feature Deep-Dive

Progressive Disclosure

Most fine-tuning tools force every user into the same dense control panel. Lumina splits the difference: casual creators get three plain-language sliders, while engineers get a "God Mode" that exposes every raw parameter the model actually exposes.

Nothing is hidden permanently — it's layered. Toggling God Mode doesn't open a new screen; it reveals the machinery underneath the same controls, so the mental model never breaks.

Plain-language defaults
"Creativity" and "Response length" map to sensible parameter combinations out of the box.
God Mode for engineers
Raw temperature, top_p, top_k, and logit bias maps become editable the instant they're needed.
One state, two views
Both modes read and write the same underlying config — switching never resets a session.
